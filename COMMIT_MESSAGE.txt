Fix: MySQL prepared statements parameter binding for positional placeholders

Исправлена проблема с биндингом параметров при использовании позиционных
плейсхолдеров (?) в prepared statements.

Проблема:
PDO требует, чтобы параметры для позиционных плейсхолдеров передавались
в массиве с последовательными числовыми индексами (0, 1, 2, ...).
При передаче ассоциативного массива со строковыми ключами или несвязными
индексами возникала ошибка: SQLSTATE[HY093]: Invalid parameter number

Решение:
- Добавлен метод usesPositionalPlaceholders() для определения типа плейсхолдеров
- Модифицирован метод prepareAndExecute() для автоматической нормализации
  параметров через array_values() при использовании позиционных плейсхолдеров
- Именованные параметры (:name) работают без изменений

Изменения:
- src/MySQL.class.php: добавлен метод usesPositionalPlaceholders(), 
  обновлен метод prepareAndExecute()
- tests/Unit/MySQLPreparedStmtBindingTest.php: 23 unit теста
- CHANGELOG_MySQL_PreparedStmt.md: подробная документация
